# Order Placement

The Dealing module is where the trading desk turns approved investments into real-world orders. It handles order placement, allocation across portfolios, commission adjustments, broker grouping and X-Transaction approvals.

### Order Placement

Order Placement is used to manage and track every investment order. From this screen the dealer can place new orders, halt or resume them, and follow each order’s status from creation through to confirmation.



| **Component**          | **Description (English)**                                               | **คำอธิบาย (ภาษาไทย)**                                         |
| ---------------------- | ----------------------------------------------------------------------- | -------------------------------------------------------------- |
| All Unexpired Orders   | Display only active orders that have not yet reached their expiry date. | แสดงเฉพาะคำสั่งซื้อขายที่ยังมีผลอยู่ (ยังไม่หมดอายุ)           |
| Create / Trade Date    | Toggle filtering between the date the order was created vs. traded.     | สลับการกรองตามวันที่สร้างคำสั่ง หรือวันที่ส่งคำสั่งซื้อขายจริง |
| Date Filter            | Select a specific date or range to view the order history.              | เครื่องมือเลือกวันที่เพื่อเรียกดูรายการคำสั่งซื้อขาย           |
| Columns                | Customize and toggle the visibility of specific data columns.           | ปรับเปลี่ยนคอลัมน์ข้อมูลที่ต้องการให้แสดงผลบนหน้าจอ            |
| Grouping               | Organize orders into categories (e.g., by Broker, Side, or Security).   | จัดกลุ่มคำสั่งซื้อขายตามหมวดหมู่ที่ต้องการ                     |
| Order Placement Table  | The main table listing all investment orders and their details.         | ตารางหลักที่รวบรวมรายการคำสั่งซื้อขายทั้งหมดในระบบ             |
