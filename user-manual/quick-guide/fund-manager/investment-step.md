# Investment Step —

This end-to-end walkthrough demonstrates how an investment travels from creation through Order Placement, Allocation, Post Allocation, Post Transaction and Process EOD.

#### 7.14.1 Create a New Investment

From the left Navbar, navigate to Investment ▸ Invest.

1\.       Click (1) to create a new investment.

2\.       Enter an Investment Name (2).

3\.       Click (3) to confirm.

Figure 7.14a — Invest interface and New Investment pop-up

4\.       Choose a Portfolio Model (4).

5\.       Click (5) to confirm and create the investment.

#### 7.14.2 Edit Investment — Matrix

A Portfolio Investment Adjustment window appears. Use it to modify investment details.

<figure><img src="../../.gitbook/assets/ชื่อเรื่อง Page 132 - คำอธิบาย Figure 7.14c — Edit Investment (Matrix)" alt=""><figcaption></figcaption></figure>

Figure 7.14c — Edit Investment (Matrix)

**Toolbar reference**

A.  Undo / Redo Action: Reverse or reapply the most recent changes.

B.  Percentage: View / enter values as a percentage.

C.  Unit: View / enter values as units.

D.  Value: View / enter values as money.

E.  Reference values: Toggle the display of reference values on each cell.

F.  Column size: Increase / decrease the column width.

G.  Variation: Show the variation / change orders.

H.  Main Currencies target: Investment target of an entire Main Currencies foldout group.

I.  Conform all portfolios: Conform every portfolio to the model.

J.  Investment Group target: Investment target of the Investment Group foldout.

K.  Conform foldout group: Conform every security in the foldout group.

L.  Conform this row: Conform the selected security across all portfolios.

M.  Batch Set Percentage: Set the same percentage for many cells at once.

N.  Validation: You cannot submit while any ERROR-severity issue remains.

8\.       Press Enter to confirm.

#### 7.14.5 Orders tab

In the Orders tab you can:

10\.   View Orders and Details.

11\.   Use button (6) to download a reference file for importing orders.

12\.   Use button (7) to upload the prepared .XLSX file.

<figure><img src="../../.gitbook/assets/ชื่อเรื่อง Page 136 - คำอธิบาย Figure 7.14f — Edit Investment (Orders)" alt=""><figcaption></figcaption></figure>

Figure 7.14f — Edit Investment (Orders)

**How the system computes orders**

The system automatically calculates stock transaction values using the Market Price. When selling stocks, the available funds from the sale are calculated; if there is also a buy order, the system prioritises using those funds. If the proceeds are not enough, the system pulls additional cash to complete the purchase.

Click (8) to confirm the changes.

#### 7.14.6 Resolve issues before submission

If the Submit button is disabled, open the Issues section and click the notification (9) for details. The Issues window will show every error that needs to be corrected.

<figure><img src="../../.gitbook/assets/ชื่อเรื่อง Page 138 - คำอธิบาย Figure 7.14h — Issues window" alt=""><figcaption></figcaption></figure>

Figure 7.14h — Issues window

#### 7.14.7 Submit with approval

When the Submit button is active but an Approval notification appears, clicking Submit opens a pop-up that asks for the approval reason (10). Enter the reason and click (11) to send the request to the Compliance menu.

<figure><img src="../../.gitbook/assets/ชื่อเรื่อง Page 139 - คำอธิบาย Figure 7.14i — Approval Reason pop-up" alt=""><figcaption></figcaption></figure>

Figure 7.14i — Approval Reason pop-up

The status updates to Pending until Compliance / Risk reviews the request. If the request is denied every order is blocked.

#### 7.14.8 Approved status

Once Compliance / Risk approves, the status updates so the order can proceed.

<figure><img src="../../.gitbook/assets/ชื่อเรื่อง Page 140 - คำอธิบาย Figure 7.14j — Approved order status" alt=""><figcaption></figcaption></figure>

Figure 7.14j — Approved order status

When Submit is available with a Warning, you can still proceed; the order is sent to Order Placement for the next step.

#### 7.14.9 Revert a submission

Click (12) to view order details. Press (13) to undo every order in the submission, or click (14) on a single row to cancel just that order.

#### 7.14.10 Order Placement

From the left Navbar, navigate to Dealing ▸ Order Placement.

13\.   Click (1) to stop the selected order.

14\.   Click (2) to revert the Stop status back to Initial.

15\.   Click (3) to place the order — a pop-up appears.

<figure><img src="../../.gitbook/assets/ชื่อเรื่อง Page 142 - คำอธิบาย Figure 7.14l — Order Placement" alt=""><figcaption></figcaption></figure>

Figure 7.14l — Order Placement

The pop-up shows the transaction details. Specify Broker (2)(3) and Units (4); split across brokers if needed. The status updates to a partial-placed marker (5) until every unit is placed, at which point the status becomes complete (6).

<figure><img src="../../.gitbook/assets/ชื่อเรื่อง Page 143 - คำอธิบาย Figure 7.14m — Order Placement pop-up" alt=""><figcaption></figcaption></figure>

Figure 7.14m — Order Placement pop-up

#### 7.14.11 Allocation

Navigate to Dealing ▸ Allocation. Click (7) to allocate the order. The Create Dealing pop-up appears. Review the Trade Date, Cost Per Unit and other fields, then click (8) to confirm.

<figure><img src="../../.gitbook/assets/ชื่อเรื่อง Page 144 - คำอธิบาย Figure 7.14n — Allocation" alt=""><figcaption></figcaption></figure>

Figure 7.14n — Allocation

Press Confirm (9) to verify the order. A review pop-up appears. Click (10) to confirm and update the status.

<figure><img src="../../.gitbook/assets/ชื่อเรื่อง Page 145 - คำอธิบาย Figure 7.14o — Confirm allocation" alt=""><figcaption></figcaption></figure>

Figure 7.14o — Confirm allocation

If the order has multiple portfolios, after pressing (8) press (12) to distribute units. Then in the Allocate pop-up, press (13) to auto-allocate, and (14) to confirm.

<figure><img src="../../.gitbook/assets/ชื่อเรื่อง Page 146 - คำอธิบาย Figure 7.14p — Multi-portfolio allocation" alt=""><figcaption></figcaption></figure>

Figure 7.14p — Multi-portfolio allocation

When the “Do you want to also confirm all created transactions?” pop-up appears, press (15) to allow commission adjustments, or (16) to send the transaction directly to Post Allocation.

<figure><img src="../../.gitbook/assets/ชื่อเรื่อง Page 147 - คำอธิบาย Figure 7.14q — Confirm transactions pop-up" alt=""><figcaption></figcaption></figure>

Figure 7.14q — Confirm transactions pop-up

Navigate to Dealing ▸ Commission Adjustment. Modify the commission rate directly (17), or download an Excel file (18) for offline adjustments and upload it back later (19).

<figure><img src="../../.gitbook/assets/ชื่อเรื่อง Page 148 - คำอธิบาย Figure 7.14r — Commission Adjustment" alt=""><figcaption></figcaption></figure>

#### 7.14.13 Post Allocation

Navigate to Dealing ▸ Post Allocation. Press (20) to edit the order or (21) to confirm it.

<figure><img src="../../.gitbook/assets/ชื่อเรื่อง Page 149 - คำอธิบาย Figure 7.14s — Post Allocation" alt=""><figcaption></figcaption></figure>

Figure 7.14s — Post Allocation

After clicking (21) a pop-up appears confirming a Transaction ID. Press (22) to finalise the transaction.

<figure><img src="../../.gitbook/assets/ชื่อเรื่อง Page 150 - คำอธิบาย Figure 7.14t — Confirming Transaction ID" alt=""><figcaption></figcaption></figure>

#### 7.14.14 Post Transaction

Navigate to Dealing ▸ Post Transaction. Press (23) to post the transaction, or select multiple portfolios and press (24) to post them in bulk. Posted transactions move to the Close Transaction menu.

<figure><img src="../../.gitbook/assets/ชื่อเรื่อง Page 151 - คำอธิบาย Figure 7.14u — Post Transaction &#x26; Close Transaction" alt=""><figcaption></figcaption></figure>

Figure 7.14u — Post Transaction & Close Transaction

#### 7.14.15 Process EOD

Finally, navigate to Daily ▸ Process EOD. Select the period, choose the portfolio group or individual portfolios, move them to Selected and press (25) to complete the investment process.

<figure><img src="../../.gitbook/assets/ชื่อเรื่อง Page 152 - คำอธิบาย Figure 7.14v — Process EOD" alt=""><figcaption></figcaption></figure>
