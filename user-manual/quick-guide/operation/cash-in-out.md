# Cash In/Out —

### Cash In — Deposit Subscription

{% stepper %}
{% step %}
#### Deposit: Subscription

Navigate to Daily ▸ Cash In/Out. \
Fill in the entry fields, choose Cash TX type “Subscription”

Set Amount with the unit ratio (Unit amount = Amount / NAV of the portfolio). <br>

Click <a class="button secondary">Create</a> to create the deposit transaction.

<div data-with-frame="true"><figure><img src="../../.gitbook/assets/image (139).png" alt=""><figcaption></figcaption></figure></div>

{% hint style="info" %}
For the first time Deposit.

Unit = ratio 10:1&#x20;

Amount : 10 ,  Unit : 1
{% endhint %}
{% endstep %}

{% step %}
#### Navigate to Daily ▸ Post Transaction

Click <a class="button secondary" data-icon="check"></a> to post the transaction.

<div data-with-frame="true"><figure><img src="../../.gitbook/assets/image (140).png" alt=""><figcaption></figcaption></figure></div>
{% endstep %}

{% step %}
#### Navigate to Daily ▸ Process EOD

Select the relevant Portfolio and click <a class="button secondary">Process EOD</a> to execute the EOD.

<div data-with-frame="true"><figure><img src="../../.gitbook/assets/image (141).png" alt=""><figcaption></figcaption></figure></div>

{% hint style="warning" %}
_The first EOD period must match the Portfolio Start Date and must be the first cash-in date. Verify in Resources ▸ Portfolio ▸ Portfolio._
{% endhint %}
{% endstep %}
{% endstepper %}

***

### Cash Out — Withdraw Redemption

{% stepper %}
{% step %}
### Withdraw: Redemption

Navigate to Daily ▸ Cash In/Out. Choose Cash TX type “Redemption”.&#x20;

Click <a class="button secondary">Create</a> to add the withdraw transaction.

<div data-with-frame="true"><figure><img src="../../.gitbook/assets/image (143).png" alt=""><figcaption></figcaption></figure></div>
{% endstep %}

{% step %}
#### Navigate to Daily ▸ Post Transaction

Click <a class="button secondary" data-icon="check"></a> to post the transaction.

<div data-with-frame="true"><figure><img src="../../.gitbook/assets/image (144).png" alt=""><figcaption></figcaption></figure></div>
{% endstep %}

{% step %}
#### Navigate to Daily ▸ Process EOD

Select the relevant Portfolio and click <a class="button secondary">Process EOD</a> to execute the EOD.

<div data-with-frame="true"><figure><img src="../../.gitbook/assets/image (141).png" alt=""><figcaption></figcaption></figure></div>
{% endstep %}
{% endstepper %}

{% hint style="warning" %}
**Data Accuracy**

Please ensure that the End-of-Day (EOD) process is performed on the same date the transaction is initiated.
{% endhint %}

{% hint style="info" %}
**System Update**

The system will only update and reflect the latest transaction data in the portfolio reports once the EOD for that specific date is completed.
{% endhint %}
