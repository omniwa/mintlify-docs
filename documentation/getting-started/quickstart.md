### Currency Details

<details>

<summary>Currency Details</summary>

| **Component**         | **Description (English)**                                   | **คำอธิบาย (ภาษาไทย)**                                                         |
| --------------------- | ----------------------------------------------------------- | ------------------------------------------------------------------------------ |
| Day Basis             | Day count convention used for fixed-income calculation.     | เกณฑ์การนับจำนวนวันในหนึ่งปี ใช้สำหรับการคำนวณตราสารหนี้ (รองรับรายงานในอนาคต) |
| Price Scale           | Number of decimal places displayed in the system.           | จำนวนหลักของทศนิยมที่แสดงผลในระบบ (0 = ไม่มีทศนิยม, 2 = ทศนิยม 2 ตำแหน่ง)      |
| Used to calculate ZDF | Enable this currency for Zero Discount Factor calculations. | เลือกใช้สกุลเงินนี้ในการคำนวณ ZDF (สำหรับระบบงาน BOND ในอนาคต)                 |

{% hint style="info" %}
#### Note: Price Scale Functionality

The Price Scale setting is primarily used for rounding precision during the End of Day (EOD) process.
{% endhint %}

| **Feature**       | **Primary Purpose**       | **Impact**                                                                                                     |
| ----------------- | ------------------------- | -------------------------------------------------------------------------------------------------------------- |
| Price Scale       | EOD Calculation & Storage | Determines the actual mathematical precision of prices stored in the database after the End of Day process.    |
| Report Formatting | Data Presentation         | Controls how numbers appear to the user in various reports (Visual only), independent of the stored precision. |

{% hint style="info" %}
#### System Precision vs. Report Formatting

Note: The Price Scale setting in the currency configuration is strictly for data precision and rounding during the EOD process.

> Please note that rounding in reports is a separate visual formatting layer. The system may display values in reports with a different number of decimals for presentation purposes, which does not affect the underlying price scale used for system-wide calculations.
{% endhint %}

</details>
